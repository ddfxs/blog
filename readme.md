# 纯手写硬核流博客  
纯手写硬核流博客（Vanilla Web Blog）拒绝了任何复杂的打包工具和多余的框架。你的核心目标是回归 Web 的本质：只用纯 HTML 建立结构、纯 CSS 渲染视觉、纯 JS 处理微交互。

因为没有自动化框架帮你拼装页面，你需要精心设计一个低维护成本、易于手动扩展的轻量级目录大纲。   

---
# 1. 完整文件版图一览  

my-blog/                   你的项目根目录  
│
├── index.html             博客首页：展示自我介绍，动态异步抓取并渲染文章列表   
├── about.html             关于我页：展示个人履历、精致的社交媒体图标、留存留言板   
│
├── css/
│   └── style.css          视觉大脑：控制全局字体、黄金阅读宽度、自适应系统暗黑模式   
│
├── js/
│   └── main.js            组件大脑：负责在所有页面动态注入完全相同的导航栏和页脚   
│
├── data/
│   └── archive.json       数据中心：你所有文章的索引表。首页列表完全由它驱动

# 2. 它们是如何协同工作的？  
  这套系统的精妙之处在于“数据驱动”与“动态拼接”，帮你省去了传统手写站点的重复劳动  
- 访问首页： 当用户打开 index.html 时，网页底部的脚本会悄悄向 data/archive.json 发起请求，把里面的文章标题、日期、路径读取出来，自动拼成列表展示给用户。
- 统一导航： 无论用户是在首页，还是在深层的 posts/ 某篇文章里，页面加载时 js/main.js 都会自动在顶部塞入统一的导航栏。这意味着你以后想多加一个菜单，只需要改 main.js 这一处，全站同步生效。

# 3. 日常保姆级使用指南  
    配置好 Cloudflare Pages 之后，你的日常维护只需要遵循以下简单的机械化操作：  
   ##  ✍️ 情况 A：我想写一篇新文章
  -   复制模板： 进入 posts/ 目录，复制 template.html，重命名为你喜欢的名字（例如 2026-07-12-my-story.html）。
   -  编写内容： 打开它，修改 <title> 标签，并在 <section class="post-content"> 区域内直接写你的文章（用标准的 <h2>, <p>, <img> 标签）。
  -  登记入库（最关键）： 打开 data/archive.json，在 "articles" 数组的最前面追加一条你的文章信息：
-   {
-  "title": "我的新文章标题",
-  "date": "2026-07-12",
-  "path": "posts/2026-07-12-my-story.html",
-  "tags": ["生活", "随笔"]
-  }
-  提交代码： 将修改后的项目推送到 GitHub。Cloudflare Pages 会在 2 秒内捕捉到更新，你的新文章就会直接出现在博客首页上！

##  📸 情况 B：我想在文章里放张照片或音频  
   -  把你的照片（如 photo.webp）丢进 media/images/ 文件夹。 
   -  在你的文章 HTML 里，直接通过相对路径引用它： 
  -  <img src="../media/images/photo.webp" alt="照片描述">  
   -  同理，音频丢进 media/audio/，使用 <audio src="../media/audio/music.mp3" controls></audio> 引用。

  ##  🌐 情况 C：我想换个社交媒体链接  
-  直接打开 about.html，找到 <div class="social-links"> 区域，修改对应的 href="你的最新链接"，或者直接复制整段 <a href="...">...</a> 来增加新的平台图标。

  这套架构既保留了纯手写的极致轻量和极速秒开（没有一行多余代码），又通过 JSON 和 JS 实现了类似后台管理系统的便利性，可以说是独立网页开发者的终极浪漫！
